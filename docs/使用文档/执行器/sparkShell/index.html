SparkShell

简介​

我们新增了 Spark Shell 任务类型， 用户可在中台选择同源数据库， 在线编辑 Spark Scala 代码， 支持同源数据复杂操作， 不再像 Dolphinscheduler 原生的 Spark 任务一样需要重复编译代码提交 Jar 包的繁琐流程。spark shell 任务的本质是在后台调用了 spark-submit 脚本来启动应用程序的， 通过接受中台传递的数据源连接信息， 在 spark shell 任务中内置了初始化的SparkContext 对象， 利用 Spark Repl 的技术， 动态执行用户编辑的 scala 代码。在 SparkShell 的执行引擎内部， 我们封装了 ApiProxy 的 API 方便中台的用户读写mysql 和 Doris 的数据， 禁用了原始 Dataframe Read 和 Save 的 Api。 在 ApiProxy内部， 会校验该执行任务的用户是否具备访问数据的权限， 以实现任务运行时的数据安全。

使用​

SparkShellExecutor datasource(base64) code(base64)"
